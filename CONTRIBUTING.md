# Contributing

Thanks for considering a contribution.

## What's Helpful

- Bug fixes or corrections to existing examples
- Additional patterns you've found effective
- Improvements to documentation clarity
- New diagnostic queries

## Guidelines

**Code examples should be:**
- Tested in a real Snowflake environment
- Commented to explain the "why" not just the "what"
- Generic enough to adapt (no proprietary field names)

**Keep the tone:**
- Practical over theoretical
- Specific over vague
- Concise over comprehensive

## Process

1. Fork the repo
2. Create a branch for your change
3. Make your changes
4. Submit a PR with a clear description of what you changed and why

## Questions?

Open an issue if you're unsure whether something fits.
